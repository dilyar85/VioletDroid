Violetdroid

Android Application for Drawing UML Diagrams

View project on GitHub

Introduction

VioletDroid is an Android app that allows users to draw UML diagrams on their phones and tablets.

Android Framework

  • VioletDroid uses Activity - Fragment template to accomplish its major feature.
  • RecyclerView with corresponding adapter displays the elements needed in UML diagrams.
  • Static methods in the Utility class to help implement its feature and avoid duplicated codes.

Design Patterns

VioletDroid uses two major design patterns: Decorator Pattern, Composite Pattern, and Observer Pattern.

  • Decorator Pattern：
    RecyclerAdapter class extends Android’s native RecyclerView.Adapter to display the icons in the toolbar menu as a scrolling dataset of images. This is an example of the decorator pattern used in the Android API—RecyclerView essentially decorates an Android ViewGroup to provide the additional scrolling functionality and link to a dataset of assets for the view. The following table represents this pattern: ![decorator pattern image] url

  • Composite Pattern：
    The composite pattern is used in the VioletDroid CanvasLayout class which extends Android’s native RelativeLayout class, which extends the ViewGroup abstract class which extends View. So CanvasLayout is a ViewGroup, which is a View that can contain other Views. The following table represents this pattern in our project: ![composite pattern image] url

  • Observer Pattern：
    The observer pattern is another very common occurrence in User Interface design, particularly with Graphical User Interfaces, since the signal for the application to do something comes from the user’s interaction with elements on-screen, those elements must somehow notify the application what has happened - hence the listeners involved in an observer pattern: ![observer pattern image] url

Service Framework

We utilized a cloud service framework in our project to facilitate user account maintenance and saving of diagrams. We mention it here because it provides our project with a huge suite of services that we didn’t actually have to code ourselves, but merely import com.avos.avoscloud.AVOSCloud; and add 3 lines of initialization for the framework at app startup.

Run Demo

  1. Start your emulator.
  2. Via bash, navigate to the VioletDroid project folder.
  3. Execute:~> ./exec.sh
